1. 1. What is the difference between invoking and creating a function?

    2. When invoking a function, the function is executed or called to perform its task. Creating a function involves defining the function's code and behavior but does not necessarily execute it.

  2. 2. Explain the syntax regarding semicolons and code blocks.

    3. Semicolons are required after a line of code to signify that the current statement has ended. Code blocks are enclosed in curly braces {} to group multiple statements together and do not require semicolons at the end.

  3. 3. What happens when we try to store values from a function when it doesn't return any?

    4. When we try to store values from a function that doesn't return anything, the result is undefined because the function does not provide a value to be stored.

  4. 4. How do we separate parameters from each other when creating a function?

    5. We separate parameters from each other using commas when defining a function.

  5. 5. Explain how parameter names are related during the invocation and creation of a function.

    6. Parameter names can be different during invocation and creation, but the order and number of arguments must match the parameters defined in the function.

  6. 6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. When a function call is missing a parameter, the corresponding parameter in the function definition is assigned the value undefined because no argument was provided for it.

  7. 7. Explain what happens when we use recursive functions.

    8. Recursive functions call themselves in order to solve smaller instances of the same problem, typically until a base case is reached to stop the recursion.
